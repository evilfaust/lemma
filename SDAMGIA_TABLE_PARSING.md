# Парсинг таблиц с РЕШУ ЕГЭ (sdamgia.ru)

## Обзор

PDF-сервис (`pocketbase/pdf-service.js`) теперь поддерживает **автоматическую конвертацию HTML-таблиц в Markdown** при импорте задач с сайта sdamgia.ru.

## Что изменилось

### Функция `processCondition`

Добавлена обработка HTML таблиц:

1. **Распознавание таблиц** — находит все `<table>` элементы в условии задачи
2. **Обработка формул в таблицах** — LaTeX формулы из SVG alt-текстов корректно встраиваются в ячейки
3. **Конвертация в Markdown** — таблицы преобразуются в формат GFM (GitHub Flavored Markdown)

### Новая функция `tableToMarkdown`

Конвертирует HTML таблицу в Markdown формат:

```javascript
function tableToMarkdown($, tableEl) {
  // Извлекает все строки (tr)
  // Для каждой строки извлекает ячейки (th, td)
  // Формирует Markdown таблицу:
  // | Заголовок 1 | Заголовок 2 |
  // | --- | --- |
  // | Данные 1 | Данные 2 |
}
```

## Рендеринг в интерфейсе

Компонент `MathRenderer.jsx` уже поддерживает:

- ✅ Markdown таблицы через `remark-gfm`
- ✅ LaTeX формулы в ячейках таблиц
- ✅ Стилизация таблиц (границы, фон заголовков)

### Кастомные компоненты для таблиц

```javascript
components: {
  table: ({ children, ...props }) => (
    <table {...props} style={{
      borderCollapse: 'collapse',
      width: '100%',
      marginBottom: '1em',
      border: '1px solid #ddd'
    }}>
      {children}
    </table>
  ),
  thead: ({ children, ...props }) => (
    <thead {...props} style={{ backgroundColor: '#f5f5f5' }}>
      {children}
    </thead>
  ),
  td: ({ children, ...props }) => {
    // Обработка LaTeX формул в ячейках
    if (typeof children === 'string') {
      return <td {...props}>{processLatex(children)}</td>;
    }
    return <td {...props}>{children}</td>;
  },
  th: ({ children, ...props }) => {
    // Обработка LaTeX формул в заголовках
    if (typeof children === 'string') {
      return <th {...props}>{processLatex(children)}</th>;
    }
    return <th {...props}>{children}</th>;
  }
}
```

## Использование в UI

1. Откройте **"Импорт задач"** (TaskImporter)
2. Выберите вкладку **"Импорт с сайта"**
3. Вставьте URL с `&print=true`:
   ```
   https://mathb-ege.sdamgia.ru/test?category_id=231&filter=all_a&print=true
   ```
4. Укажите метаданные (тема, подтема, сложность, теги)
5. Нажмите **"Загрузить задачи"**
6. В предпросмотре проверьте, что таблицы корректно отображаются
7. Импортируйте задачи в базу

## Технические детали

### Порядок обработки в `processCondition`

1. **Обработка формул в таблицах** — заменяем SVG формулы на маркеры `___FORMULA_N___` внутри `<table>`
2. **Конвертация таблиц** — преобразуем `<table>` в Markdown и заменяем на маркеры `___TABLE_N___`
3. **Обработка изображений вне таблиц** — извлекаем обычные изображения и формулы вне таблиц
4. **Извлечение текста** — получаем текст с маркерами
5. **Замена маркеров** — восстанавливаем таблицы, формулы, изображения в правильном порядке

### Поддерживаемые элементы

- ✅ `<table>`, `<tr>`, `<td>`, `<th>` — стандартные таблицы
- ✅ SVG формулы в ячейках (через `alt` атрибут)
- ✅ Многострочные заголовки
- ✅ Пустые ячейки
- ✅ Таблицы с colspan/rowspan (отображаются как обычные ячейки)

## Ограничения

- **Colspan/rowspan** — сложные объединения ячеек упрощаются (Markdown не поддерживает colspan)
- **Вложенные таблицы** — обрабатывается только верхний уровень
- **Стили CSS** — цвета, размеры шрифтов теряются (сохраняется только структура)

## Возможные улучшения

1. **Обработка сложных таблиц** — добавить поддержку colspan через дублирование ячеек
2. **Сохранение выравнивания** — Markdown поддерживает выравнивание через `:---`, `---:`, `:---:`
3. **Парсинг других сайтов** — адаптировать логику для других источников задач
