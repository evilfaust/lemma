# Changelog — История изменений

## [3.6.5] - 2026-02-24

### Добавлено
- **Изображения в обычных задачах** — `TaskEditModal` теперь поддерживает 3 режима:
  - **По ссылке** — ввод URL (как раньше)
  - **Загрузить** — загрузка PNG/JPG с компьютера + кадрирование
  - **Нарисовать** — встроенный GeoGebra-апплет → экспорт PNG → кадрирование
- **`GeoGebraDrawingPanel`** — новый компонент: упрощённая панель GeoGebra без сохранения состояния (только PNG-результат). Кнопки: «Сохранить PNG», «Обрезать», «Очистить».
- **`utils/cropImage.js`** — утилиты обрезки PNG вынесены в shared-модуль: `cropPngByMargins`, `normalizeCrop`, `dataUrlToFile` и др.

### Улучшено
- **GeoGebraApplet**: polling через `requestAnimationFrame` (до 180 кадров) заменён на `ResizeObserver` — ноль принудительных reflow, event-driven подход.
- **GeometryTaskEditor**: убраны каскадные fallback'ы при сохранении (до 12 HTTP-запросов → 1). Схема зафиксирована по реальной БД на VPS.

### Рефакторинг
- `GeometryTaskEditor` — локальные утилиты обрезки заменены на импорт из `utils/cropImage.js`.

## [3.6.4] - 2026-02-23

### Добавлено
- **Геометрия: переключаемый список задач** — режимы `Таблица` и `Карточки` в `GeometryTaskList`.
  В карточках доступны: выбор задач, действия (редактировать/просмотр/удалить), превью PNG-чертежа,
  LaTeX-ответ и пагинация `10/20/50`.

### Изменено
- **Предпросмотр листа геометрии (`GeometryTaskPreview`)**:
  - drag&drop смены порядка задач доступен прямо в режиме просмотра (когда выключено `Редактировать макет`);
  - нумерация карточек сделана сквозной по позиции в листе (`1..N`), без привязки к коду задачи.

### Документация
- Добавлен файл **`SCHEMA_GEOMETRY.md`** как актуальный source of truth по коллекциям геометрии на production:
  типы полей, обязательность, правила сохранения PNG/GeoGebra и проверка схемы.

## [3.6.3] - 2026-02-23

### Исправлено
- **Синхронизация схемы `geometry_tasks` с реальным VPS (`task-ege.oipav.ru`)**:
  - поле `preview_layout` добавлено в коллекцию и таблицу;
  - поле `topic` восстановлено и переведено в `relation → geometry_topics`;
  - поле `subtopic` приведено к `relation → geometry_subtopics`;
  - пустые `topic` заполнены по выбранным `subtopic`.
- **`task_type` больше не блокирует сохранение**:
  - в схеме VPS снят `required`, существующие пустые значения заполнены `ready`;
  - фронтенд-сохранение использует валидные значения enum (`ready|build|mixed`) при fallback.
- **PNG-чертеж в геометрии хранится как файл**:
  - поле `geogebra_image_base64` приведено к типу `file` (PocketBase storage);
  - предпросмотры/списки используют URL файла из storage.

### Документация
- Обновлены `README.MD` и `CLAUDE.MD` под фактическую схему геометрического модуля:
  `topic/subtopic` relations, `preview_layout`, `task_type` enum, `geogebra_image_base64` как file field.

## [3.6.2] - 2026-02-22

### Добавлено
- **Быстрый предпросмотр задачи** прямо из списка: модальное окно с `GeometryPreviewCard`,
  переключатель «Показывать ответ», режим редактирования макета (drag+resize) и сохранение
  макета в `preview_layout` — всё без перехода в полный предпросмотр.
- **Скрипт миграции** `scripts/migrate-geometry-images.mjs`: перенёс 17 PNG-чертежей
  из текстового поля `geogebra_image_base64` в файловое поле PocketBase `drawing_image`.

### Исправлено
- **Критический баг редактора**: при открытии задачи из списка `statement_md`, `solution_md`
  и `geogebra_base64` не попадали в редактор (не входят в `LIGHT_FIELDS`). Редактор показывал
  пустые поля и затирал данные при сохранении. Исправлено: `openEdit()` теперь вызывает
  `getGeometryTask(id)` (полная запись) перед открытием редактора.
- **GeoGebra-аплет загружался пустым** при редактировании — следствие того же бага.
  После исправления `geogebra_base64` (XML-состояние `.ggb`) корректно загружается в аплет.
- **Масштабирование в быстром предпросмотре**: `mode="print"` использовал `mm`-единицы,
  которые не масштабируются с контейнером. Заменено на `mode="student"` (px).
- **Клиппинг чертежа**: убраны `max-height: 45mm` и `max-height: 220px` из CSS.
- **Колонка «Чертёж»** в списке: `geogebra_base64` не входит в LIGHT_FIELDS,
  поэтому значок GGB определяется через `drawing_view === 'geogebra'`.

### Изменено
- **PNG-хранилище**: чертежи перенесены из base64 TEXT-поля в файловое поле `drawing_image`
  (FileField, max 10 МБ). Преимущества: −33% размера, HTTP-кеширование, нет в API-ответах списка.
  - `getGeometryImageUrl(task)` — хелпер URL файла (fallback: legacy base64).
  - `LIGHT_FIELDS` для `getGeometryTasks()`: исключает тяжёлые поля (geogebra_base64, solution_md),
    `statement_md` включён (нужен для быстрого предпросмотра).
  - `GeometryTaskEditor.handleSave()`: PNG → `File` через `atob()`, сохраняется в `drawing_image`.
- **Индикатор загрузки** на кнопке «Редактировать» в строке таблицы (`loading` по `task.id`).

### Удалено
- **Поле `task_type`** из `geometry_tasks` (DB + фронтенд).
  Значения «Готовый чертёж / Построение / Смешанный» не имели практического смысла:
  режим «Построение» требует живого GeoGebra-аплета на стороне студента и автоматической
  проверки результата, что не реализовано и не планируется.
  Геометрический модуль сфокусирован на сборе задач для печатных листов A5.
  - Удалены: `TASK_TYPE_OPTIONS`, `TASK_TYPE_LABELS`, фильтр, колонка таблицы
  - Миграция: `1771900000_remove_task_type_from_geometry_tasks.js` (применена на VPS)

## [3.6.0] - 2026-02-20

### Добавлено
- **Раздельные Vite-приложения** — монорепо с двумя независимыми бандлами из одного репозитория
  - `index.html` + `src/teacher/main.jsx` — учительское приложение (локально)
  - `student.html` + `src/student/main.jsx` — студенческое приложение (VPS)
  - `npm run build` собирает оба бандла одновременно
- **`src/shared/`** — общий код для обоих приложений
  - `shared/services/pocketbase.js` + `pocketbaseUrl.js` — API и URL-резолвер
  - `shared/components/MathRenderer.jsx` — рендерер LaTeX + Markdown
  - `shared/utils/shuffle.js` + `escapeFilter.js` — утилиты
  - Оригинальные пути сохранены как реэкспорты (обратная совместимость)
- **`student.oipav.ru`** — отдельный домен для студенческого интерфейса
  - Let's Encrypt SSL сертификат
  - nginx конфиг на VPS: `/var/www/student-ege/`
  - Фронтенд задеплоен: `https://student.oipav.ru`
- **`VITE_STUDENT_URL`** — переменная окружения для домена студентов
- **`raspberry/deploy-student-to-vps.sh`** — скрипт деплоя студенческого приложения на VPS
- Alias `@shared` в `vite.config.js` для удобного импорта shared-модулей

### Изменено
- **`SessionPanel.jsx`** — `buildStudentUrl()` использует `VITE_STUDENT_URL` вместо `window.location.origin`
  - QR-коды и ссылки теперь ведут на `https://student.oipav.ru/student/{sessionId}`
- **`raspberry/nginx.conf`** — обновлён для поддержки двух SPA
  - `/student/` → `student.html`
  - `/` → `index.html`
- **`ege-tasks/index.html`** — entrypoint учительского приложения → `src/teacher/main.jsx`
- **`ege-tasks/vite.config.js`** — multi-input build + `@shared` alias
- Удалён `src/main.jsx` (заменён двумя специализированными entrypoints)
- Обновлён деплой Raspberry Pi (nginx конфиг + оба HTML файла)

### Архитектура
```
Учитель (локально):           Студент (VPS: student.oipav.ru):
index.html                    student.html
src/teacher/main.jsx    →     src/student/main.jsx
App.jsx + все компоненты      StudentApp.jsx + student/ компоненты
         ↓                              ↓
         └──── src/shared/ ────────────┘
               (pocketbase, MathRenderer, shuffle, escapeFilter)
                        ↓
               PocketBase (task-ege.oipav.ru)
```
