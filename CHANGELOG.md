## [3.9.2] - 2026-04-11

### Шифровки — новый инструмент для уроков

**Новый раздел «Шифровки»** в меню учителя:

- **`CryptogramGenerator.jsx`** — главный компонент: левая панель (название, фраза, задачи) + правая панель (превью таблицы-шифра и строки ответа)
- **`CryptogramGenerator.css`** — стили для экрана и печати (A5 portrait, двухколоночный список задач)
- **`utils/cryptogram.js`** — логика шифровки: `buildCryptogramForVariant`, `getCryptogramLetterCount`, `getCryptogramUniqueLetterCount`, `normalizeCryptogramPhrase`

**Ключевая логика:**
- Количество задач = число **уникальных** букв в фразе (одна задача → одна буква, повторения используют тот же ответ)
- Над клетками строки ответа — **порядковый номер позиции** (1, 2, 3…), не ответ — ученик не видит слово, пока не решит
- Обманки — только целые числа и конечные десятичные дроби (без LaTeX-дробей и корней)
- Пробелы между словами создают наглядные разрывы в строке ответа

**Подбор задач:**
- **Автогенератор** — выбор темы и подтемы, кнопка «Сгенерировать (N)» подтягивает задачи с ответами из каталога
- **Ручной выбор** — через `TaskSelectModal` с возможностью переупорядочить и удалить

**Параметры:**
- Переключатель «Срезать «Вычислите», «Найдите» и т.д.» (использует `filterTaskText`, включён по умолчанию)
- Фраза «Найдите частное от деления» добавлена в `filterTaskText.js`

**Печать (A5 portrait):**
- Инструкция ученику: «Найди свой ответ в таблице — запиши соответствующую букву»
- Страница-ключ учителя (после page-break) с правильной фразой
- Scoped print CSS через `body:has(.cgp-root)`

## [3.8.4] - 2026-04-05

### Тригонометрия — генератор листов «Единичная окружность»

**Новый раздел «Тригонометрия»** в меню учителя:

- **`UnitCircleGenerator.jsx`** — главный компонент-генератор: панель настроек, превью вариантов, кнопки «Сгенерировать / Печать / Сохранить / Загрузить / Сброс»
- **`trig/UnitCircleSVG.jsx`** — SVG-окружность: оси со стрелками, засечки на 16 стандартных позициях, нумерованные точки, KaTeX-подписи углов через `<foreignObject>`
- **`trig/UnitCirclePrintLayout.jsx`** + **`UnitCirclePrintLayout.css`** — печатная вёрстка A4: страницы учеников + страницы ответов учителя; scoped print CSS

**Два типа заданий:**
- **Прямая задача** — точки отмечены на окружности, ученик подписывает углы (сетка ответов с пустыми строками)
- **Обратная задача** — список углов задан, ученик отмечает точки на пустой окружности
- **Смешанная** — чередование прямой/обратной в рамках одного варианта

**Стандартные углы:** 16 позиций (кратные π/6, π/4, π/3, π/2). Смещение ±k×2π (k до ±4), что даёт углы вида 13π/6, −11π/6, 25π/6 и т.д.

**Настройки:**
- Количество вариантов (1–30)
- Кружков на странице A4: 2 (один под другим) или 4 (сетка 2×2)
- Точек на кружок (4–12)
- Максимальное смещение k×2π (0–4)
- Подписи на окружности: ничего / только оси / все 16 позиций
- Показывать градусы (0°, 90°, 180°, 270°)
- Засечки на 16 стандартных позициях
- Лист ответов учителя

**LaTeX-дроби:** все углы рендерятся через KaTeX — `\dfrac{13\pi}{6}`, `-\dfrac{11\pi}{6}` и т.д. В SVG через `<foreignObject>`, в HTML через `dangerouslySetInnerHTML`.

**База данных:** новая коллекция `unit_circle_worksheets` (миграция `1772000012`); поля: title, task_type, variants_count, settings(json), tasks_data(json). Superuser only.

**Хук `useUnitCircle`:** генерация вариантов, управление настройками, CRUD (getUnitCircleWorksheets / createUnitCircleWorksheet / updateUnitCircleWorksheet / deleteUnitCircleWorksheet).

## [3.8.0] - 2026-03-28

### QR-листы — новый инструмент для уроков

Полностью новый инструмент генерации обучающих листов с QR-кодом. Ученик решает задачи, находит числа-ответы в таблице и закрашивает клетки — из них складывается QR-код, который отсканировать = следующее задание.

**Компоненты:**
- **QRWorksheetGenerator** — главный компонент (настройки, управление задачами, предпросмотр, действия)
- **QRTaskPanel** — панель выбора задач: TaskSelectModal, поля ответов (InputNumber), кнопки редактирования/удаления/перемещения
- **QRGridPreview** — экранный превью сетки с режимами «ученик» / «учитель» (подсветка правильных клеток)
- **QRPrintLayout** — печатная A4-вёрстка: страница ученика + опциональная страница-ключ учителя (ответы + QR-код + подсвеченная сетка)

**Утилиты:**
- `qrMatrix.js` — генерация QR-матрицы из URL через библиотеку `qrcode`, оценка размера `estimateQRSize`
- `qrGridFill.js` — заполнение числовой сетки: правильные ячейки = модули QR, остальные — случайные числа из диапазона ответов
- `useQRWorksheet.js` — хук управления состоянием (tasks, customAnswers, qrUrl, matrix, grid, mode, title, generate, reset, loadFromSaved, updateTask)

**Сохранение и загрузка:**
- Новая коллекция `qr_worksheets` в PocketBase (title, qr_url, tasks relation, custom_answers JSON, grid JSON, qr_size)
- Кнопки «Сохранить» и «Загрузить» — список сохранённых листов с возможностью удаления
- Загрузка восстанавливает полный порядок задач из `expand.tasks`

**Интеграция с «Моими работами»:**
- Выпадающий список работ (ленивая загрузка при первом открытии)
- Выбор работы → автоматически создаётся/получается сессия → подставляется URL для студента

**Сокращение URL (clck.ru):**
- Кнопка «Сократить» рядом с полем URL
- Использует открытый API `https://clck.ru/--?url=...` (CORS разрешён, без прокси)
- Кнопка отключается, если URL уже сокращён (содержит `clck.ru`)

**Редактирование задач прямо в листе:**
- Кнопка-карандаш в каждой карточке задачи → открывается `TaskEditModal`
- Изменения сохраняются в БД и обновляются локально без перезагрузки

**Печать:**

## [3.7.9] - 2026-03-27

### Генератор вариантов ЕГЭ — КИМ-формат (EgeVariantGenerator)

Полностью переработана печать в стиле КИМ.

**Архитектура рендеринга:**
- Переход на **A5-страницы** (`@page { size: A5 portrait; margin: 0; }`) вместо A4-landscape-разворотов. Каждая страница — самостоятельный блок 148.5mm × 210mm, CSS прост и предсказуем.
- Для A4-буклета: выбрать «Страниц на листе: 2» в диалоге печати Chrome/Safari.
- `handleKimPrint` теперь инжектирует `@page { size: A5 portrait; margin: 0; }` (был A4 landscape) — Chrome корректно распознаёт формат в диалоге печати.
- `@page` вынесен на верхний уровень CSS (не внутри `@media print`) — обязательное условие для распознавания Chrome.

**Измерение высот задач:**
- Удалена проблемная функция `measureTaskHeights` (использовала `createRoot + flushSync` в оторванном от DOM элементе — CSS-правила не применялись, таблицы измерялись неверно).
- Реализован **двухфазный рендер** через `useLayoutEffect`:
  1. Фаза measure: все задачи рендерятся в скрытый контейнер **внутри React-дерева** (CSS применяется корректно — таблицы, KaTeX, изображения).
  2. `useLayoutEffect` синхронно читает `offsetHeight` каждой задачи, вычисляет пагинацию.
  3. Фаза render: отображает список A5-страниц.
- Измерительный контейнер: `width: 132.5mm` (= 148.5mm A5 − 2×8mm padding).
- Константы высот пересчитаны для A5: 180mm (обычная страница), 159mm (первая — с рамкой-примечанием).

**Удалено:** `.kim-spread`, `KimBlankPage`, `waitForImage`, `createRoot/flushSync` импорты.

- **TDFManager** — список наборов ТДФ с фильтром по классу, CRUD, переходы к редактору и вариантам
- **TDFEditor** — редактор конспекта: список пунктов с drag&drop, секции-заголовки, вставка/редактирование пунктов, кнопка печати эталонного конспекта
- **TDFItemModal** — модал создания/редактирования пункта: тип, название, вопрос (MD+LaTeX), формулировка (MD+LaTeX), краткая запись, чертёж (PNG-загрузка или GeoGebra)
- **TDFVariantBuilder** — конструктор вариантов опросников: выбор пунктов, нумерация, печать бланков
- **TDFPrintView** — печатный вид A4 landscape, два режима: `etalon` (полный конспект) и `blank` (бланк для ученика с пустыми линиями)

**Типы пунктов** (7 типов с цветовой маркировкой):
Теорема · Определение · Формула · Аксиома · Свойство · Признак · **Следствие** (новый)

**База данных** (3 новые коллекции):
- `tdf_sets` — наборы ТДФ (title, description, class_number)
- `tdf_items` — пункты (type select, name, question_md, formulation_md, short_notation_md, drawing_image file, geogebra_base64)
- `tdf_variants` — варианты опросников (number, title, item_ids JSON)

**API** (13 методов): `getTdfSets/Set/createTdfSet/updateTdfSet/deleteTdfSet`, аналогично для `Items` и `Variants`, плюс `getTdfItemDrawingUrl`

**Навигация**: 3 view в App.jsx — `tdf`, `tdf-editor`, `tdf-variants`

- **Масштабирование в быстром предпросмотре**: `mode="print"` использовал `mm`-единицы,
  которые не масштабируются с контейнером. Заменено на `mode="student"` (px).
- **Клиппинг чертежа**: убраны `max-height: 45mm` и `max-height: 220px` из CSS.
- **Колонка «Чертёж»** в списке: `geogebra_base64` не входит в LIGHT_FIELDS,
  поэтому значок GGB определяется через `drawing_view === 'geogebra'`.

### Изменено
- **PNG-хранилище**: чертежи перенесены из base64 TEXT-поля в файловое поле `drawing_image`
  (FileField, max 10 МБ). Преимущества: −33% размера, HTTP-кеширование, нет в API-ответах списка.
  - `getGeometryImageUrl(task)` — хелпер URL файла (fallback: legacy base64).
  - `LIGHT_FIELDS` для `getGeometryTasks()`: исключает тяжёлые поля (geogebra_base64, solution_md),
    `statement_md` включён (нужен для быстрого предпросмотра).
  - `GeometryTaskEditor.handleSave()`: PNG → `File` через `atob()`, сохраняется в `drawing_image`.
- **Индикатор загрузки** на кнопке «Редактировать» в строке таблицы (`loading` по `task.id`).

### Удалено
- **Поле `task_type`** из `geometry_tasks` (DB + фронтенд).
  Значения «Готовый чертёж / Построение / Смешанный» не имели практического смысла:
  режим «Построение» требует живого GeoGebra-аплета на стороне студента и автоматической
