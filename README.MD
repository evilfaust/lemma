# 🎓 EGE Tasks Manager - Система управления задачами для подготовки к ЕГЭ

## 📋 Описание проекта

**EGE Tasks Manager** — это полнофункциональное веб-приложение для управления базой задач по подготовке к ЕГЭ (Единому государственному экзамену). Система позволяет хранить, фильтровать, редактировать задачи, а также генерировать учебные карточки для печати с поддержкой математических формул LaTeX.

## 🌟 Основные возможности

### 📚 Управление задачами
- **Просмотр и фильтрация** задач по темам, сложности, источникам и годам
- **Поддержка LaTeX** для отображения математических формул
- **Редактирование задач** прямо в интерфейсе
- **Система тегов** для дополнительной классификации
- **Изображения** в условиях задач
- **Отслеживание** наличия ответов и решений
- **Пагинация** с настраиваемым количеством элементов на странице
- **Массовые операции**: теги (добавить/заменить), сложность, источник, тема, подтемы, удаление
- **Статистика и аналитика** по темам, подтемам, тегам, сложности и источникам
- **Каталог справочников**: темы/подтемы/теги/источники/прочее, CRUD и объединение дублей

### 🎲 Генератор карточек
- **Случайная выборка** задач по заданным критериям
- **Три формата печати**: A6 (4 на лист), A5 (2 на лист), A4 (1 на лист)
- **Автоматическая генерация ответов** на отдельных листах
- **Сохранение карточек** в базу данных для повторного использования
- **Управление коллекцией** сохранённых карточек
- **Настройка отображения** ответов и решений
- **Размеры шрифта**: 10px, 11px, 13px, 15px, 17px, 21px

### 📝 Генератор "Устный счет"
- **Распределение задач по тегам** с точным контролем количества
- **Компактный режим** для размещения максимального количества задач на листе
- **Grid layout** для размещения 1-3 вариантов на одной странице А4
- **Drag & Drop** - перетаскивание задач для изменения порядка внутри и между вариантами
- **Фильтр префиксов** - скрытие типовых фраз ("Вычислите:", "Найдите значение выражения" и т.д.)
- **PDF экспорт** с настраиваемыми параметрами
- **Размеры шрифта**: 10pt, 12pt, 14pt, 16pt, 20pt
- **Markdown поддержка** в условиях задач (таблицы, списки, форматирование)
- **Автоматическая прогрессия сложности** (опционально)

### 📋 Генератор контрольных работ (НОВОЕ!)
- **Конструктор структуры работы** - создание блоков задач из разных тем
- **Гибкие фильтры для каждого блока**:
  - Выбор темы (обязательно)
  - Подтемы (multiple)
  - Уровни сложности (multiple)
  - Теги (multiple)
  - Источник и год
- **Три режима генерации вариантов**:
  - Разные задачи в каждом варианте (без повторений)
  - Одинаковые задачи, разный порядок
  - Одинаковые задачи, одинаковый порядок
- **Drag & Drop** - перетаскивание задач между вариантами
- **Предпросмотр структуры** с временной шкалой блоков
- **Статистика** - количество блоков, тем и задач
- **Сохранение/загрузка работ** в базу данных
- **Редактирование сохранённых работ** (обновление вариантов и параметров)
- **Все возможности форматирования**:
  - Компактный режим
  - Настройка шрифта (10pt-20pt)
  - Количество колонок (1-3)
  - Место для решения
  - Лист с ответами
- **Экспорт в PDF** с сохранением всех настроек
- **Замена и редактирование задач** прямо в варианте
- **Автоматическая прогрессия сложности** (опционально)

### 📊 Статистика и аналитика
- **Дашборд по задачам**: темы, подтемы, теги, сложность, источники
- **Облако тегов** с быстрым переходом к задачам по тегу
- **Поиск “белых пятен”** в базе задач
- **Каталог** с быстрым переходом к задачам по фильтру и проверкой дублей

### 🖨️ Печать и экспорт
- Оптимизированная печать на формате A4
- Автоматическая разметка с разделительными линиями
- Поля для ФИО учащегося
- Прямоугольники для выписывания ответов в компактном режиме
- Экспорт в PDF с помощью html2pdf.js
- Уникальные коды для каждой задачи

### 📘 Теория + задачи
- **Связанные задачи по тегам статьи** в просмотре теории
- **Быстрый переход к задачам** из теоретического материала

## 🗄️ Структура базы данных

### Коллекция `topics` (Темы)
```javascript
{
  id: string,           // Уникальный ID
  title: string,        // Название темы (например, "Тригонометрия")
  ege_number: number,   // Номер задания в ЕГЭ (1-27)
  description: string,  // Описание темы
  order: number         // Порядок сортировки
}
```

### Коллекция `tags` (Теги)
```javascript
{
  id: string,      // Уникальный ID
  title: string,   // Название тега
  color: string    // Цвет для отображения (hex)
}
```

### Коллекция `tasks` (Задачи)
```javascript
{
  id: string,           // Уникальный ID
  code: string,         // Уникальный код задачи (например, "14-001")
  topic: relation,      // Связь с таблицей topics
  subtopic: relation[], // Массив связей с таблицей subtopics
  tags: relation[],     // Массив связей с таблицей tags
  difficulty: string,   // Уровень сложности: "1", "2", "3"
  statement_md: text,   // Условие задачи (Markdown с LaTeX)
  answer: string,       // Ответ на задачу
  solution_md: text,    // Решение (Markdown с LaTeX)
  explanation_md: text, // Дополнительные пояснения
  source: string,       // Источник задачи
  year: number,         // Год создания/публикации
  image: file,          // Изображение к задаче (опционально)
  has_image: boolean    // Флаг наличия изображения
}
```

### Коллекция `cards` (Карточки)
```javascript
{
  id: string,          // Уникальный ID
  title: string,       // Название набора карточек
  topic: relation,     // Связь с таблицей topics
  tasks: relation[],   // Массив связей с задачами
  format: string,      // Формат: "a6", "a5", "a4"
  show_answers: bool,  // Показывать ли ответы
  cards_data: json     // Данные о разбивке задач по карточкам
}
```

### Коллекция `works` (Контрольные работы)
```javascript
{
  id: string,          // Уникальный ID
  title: string,       // Название работы
  topic: relation,     // Связь с таблицей topics (опционально)
  time_limit: number,  // Лимит времени в минутах
  archived: boolean,   // Архивирована ли работа
  created: date,
  updated: date
}
```

### Коллекция `variants` (Варианты работ)
```javascript
{
  id: string,
  work: relation,      // Связь с works
  number: number,      // Номер варианта
  tasks: relation[],   // Задачи варианта
  order: json,         // Порядок задач [{ taskId, position }]
  created: date,
  updated: date
}
```

### Коллекции результатов
```javascript
work_sessions: выдача работы ученикам
attempts: попытки учеников
attempt_answers: ответы по задачам
```

## 🔧 Архитектура компонентов

### Custom React Hooks

Проект использует современный подход с переиспользуемыми хуками для разделения логики и UI:

**`useWorksheetGeneration.js`** - Универсальная логика генерации вариантов
- Поддержка структур блоков (задачи из разных тем)
- Три режима: `different` (разные), `shuffled` (перемешанные), `same` (одинаковые)
- Автоматическое исключение дубликатов
- Сортировка: по коду, сложности, случайная
- Умная обработка фильтров (тема, подтемы, теги, сложность)

**`useTaskDragDrop.js`** - Drag & Drop функционал
- Перетаскивание задач внутри варианта и между вариантами
- Визуальная индикация (dragging, drag-over)
- Поддержка сенсорных устройств

**`useWorksheetActions.js`** - Действия с работами
- Печать
- Экспорт в PDF (html2pdf.js)
- Сохранение в базу данных
- Загрузка сохранённых работ
- Удаление работ

### Переиспользуемые UI компоненты

**`worksheet/FilterBlock.jsx`** - Блок фильтров
- Выбор темы с автообновлением подтем
- Multiple select для подтем, сложности, тегов
- Опциональные фильтры (источник, год)
- Подсчёт количества задач

**`worksheet/VariantSettings.jsx`** - Настройки вариантов
- Количество вариантов (1-50)
- Режим генерации
- Тип сортировки
- Статистика

**`worksheet/FormatSettings.jsx`** - Настройки формата
- Колонки (1-3)
- Размер шрифта (10pt-20pt)
- Место для решения
- Компактный режим
- Фильтр префиксов задач

### Принципы архитектуры

1. **Разделение ответственности** - логика в хуках, UI в компонентах
2. **Переиспользование кода** - общие хуки для всех генераторов
3. **Композиция** - сборка сложных компонентов из простых
4. **Типобезопасность** - JSDoc комментарии для параметров
5. **Единообразие** - одинаковый подход во всех генераторах

## 📝 Работа с парсерами

В проекте используются **два парсера** для загрузки задач из Markdown-файлов в PocketBase. Каждый парсер работает со своим форматом файлов и папкой.

### Установка зависимостей

```bash
pip install requests pyyaml
```

---

### 1. `pb_parser.py` — Задачи из «Решу ЕГЭ» и других источников

**Папка:** `source/` (корневая)
**Файлы:** `source/14-1.md`, `source/16.md`, `source/ustno-6.md` и т.д.
**Коды задач:** `14-001`, `16-023` (формат `{номер_темы}-{порядковый}`)

#### Формат MD-файла (Решу ЕГЭ)

```markdown
---
topic: Название темы ЕГЭ
subtopic: Название подтемы
difficulty: 1
source: Решу ЕГЭ
year: 2026
tags: Вычисления
---

# Заголовок (необязательно)

**1** [1]
Условие задачи с $\LaTeX$ формулами
ответ: 42
tags: [База, Вычисления, Действия с дробями]

**2** [2]
Условие второй задачи: $\frac{a}{b} + \sqrt{x}$
ответ: $x = 5$
tags: [База, Вычисления, Степени]
```

**Описание полей:**
- `**1** [1]` — номер задачи и уровень сложности в квадратных скобках (1/2/3)
- `ответ:` — строка с ответом (с маленькой буквы)
- `tags: [...]` — теги задачи через запятую в квадратных скобках

#### Запуск

```bash
python pb_parser.py
```

Скрипт запросит в интерактивном режиме:
1. Путь к Markdown-файлу
2. Выбор темы из существующих в базе

### 2. `pb_parser_mordkovich.py` — Задачи из учебника Мордковича

**Папка:** `source/mordkovich/`
**Файлы:** `source/mordkovich/M17.md`, `source/mordkovich/M43.md` и т.д.
**Коды задач:** `M17-001`, `M43-023` (формат `M{параграф}-{порядковый}`)

#### Формат MD-файла (Мордкович)

```markdown
---
topic: Мордкович
subtopic: Логарифмические уравнения
difficulty: 2
source: Мордкович
year: null
tags: []
---

# §17. Логарифмические уравнения

**017.1a** [1]
Решите уравнение: $\log_2 x = 3$
Ответ: $x = 8$
tags: [Мордкович, Логарифмические уравнения, §17]

**017.1b** [1]
Решите уравнение: $\log_7 x = -1$
Ответ: $x = \dfrac{1}{7}$
tags: [Мордкович, Логарифмические уравнения, §17]

**017.5c** [2]
Решите уравнение: $\log_7 (x^2 - 12x + 36) = 0$
Ответ: $x = 5; \; x = 7$
tags: [Мордкович, Логарифмические уравнения, §17]
```

**Ключевые отличия от формата Решу ЕГЭ:**
- Номер задачи: `**017.1a**` — `{параграф с ведущим нулём}.{номер}{буква}` (буква: a/b/c/d)
- `Ответ:` — с **большой** буквы (важно для парсера!)
- Файл именуется `M{параграф}.md` (например, `M17.md`, `M43.md`)
- YAML: `topic: Мордкович`, `subtopic:` указывает название раздела
- Все параграфы загружаются в единый топик «Мордкович» (slug `mordkovich`)
- Подтема создаётся автоматически из поля `subtopic` в YAML

#### Запуск

```bash
# Аргумент — имя файла без расширения
python3 pb_parser_mordkovich.py M17

# Также допустимо с расширением
python3 pb_parser_mordkovich.py M17.md
```

Скрипт автоматически:
1. Извлекает номер параграфа из имени файла (M17 -> 17)
2. Находит единый топик «Мордкович» (slug `mordkovich`) — все параграфы в одном топике
3. Создаёт или находит подтему из YAML-метаданных (например, «Логарифмические уравнения»)
4. Загружает все задачи, пропуская дубликаты
5. Генерирует коды `M17-001`, `M17-002` и т.д.

## 🎯 Использование приложения

### 1. Просмотр задач

1. Откройте раздел "Все задачи"
2. Используйте фильтры:
   - **Тема** - выберите тему ЕГЭ
   - **Уровень сложности** - 1 (базовый), 2 (средний), 3 (высокий)
   - **Источник** - фильтр по источнику задач
   - **Год** - год публикации
   - **Наличие ответа/решения** - задачи с ответами или решениями
3. Настройте пагинацию (10, 20, 50 или 100 задач на странице)

### 2. Редактирование задач

1. Нажмите на кнопку "Редактировать" на карточке задачи
2. Измените нужные поля:
   - Условие (с поддержкой LaTeX)
   - Ответ (с предпросмотром)
   - Решение
3. Сохраните изменения

**Примеры LaTeX:**
- Степени: `x^2`, `a^{10}`
- Дроби: `\frac{a}{b}`
- Корни: `\sqrt{x}`, `\sqrt[3]{x}`
- Греческие буквы: `\alpha`, `\beta`, `\gamma`

### 3. Генерация карточек

1. Откройте раздел "Генератор карточек"
2. Заполните форму:
   - **Название** - имя для сохранения
   - **Тема** - выберите тему ЕГЭ
   - **Уровень сложности** - базовый, средний или высокий
   - **Количество карточек** - сколько карточек создать
   - **Заданий на карточке** - количество задач на одной карточке
   - **Формат** - A6 (4 на лист), A5 (2 на лист), A4 (1 на лист)
3. Настройте отображение:
   - **Показывать ответы** - включить/выключить ответы в карточках
   - **Показывать решения** - включить/выключить решения на листах с ответами
4. Нажмите "Сгенерировать"
5. Опционально: нажмите "Сохранить карточки" для сохранения в базу

### 4. Печать карточек

1. После генерации нажмите кнопку "Печать"
2. В диалоге печати установите:
   - **Формат бумаги:** A4
   - **Ориентация:** Книжная (Portrait)
   - **Масштаб:** 100%
   - **Поля:** Минимальные или без полей
3. Распечатайте и разрежьте по разделительным линиям

**Рекомендации:**
- Для формата A6: один лист A4 = 4 карточки
- Для формата A5: один лист A4 = 2 карточки
- Для формата A4: один лист = 1 карточка
- Ответы печатаются на отдельных листах

### 5. Управление сохранёнными карточками

1. В разделе "Генератор карточек" прокрутите вниз
2. Таблица сохранённых карточек содержит:
   - Название карточки
   - Тему
   - Количество задач
   - Формат
   - Дату создания
3. Доступные действия:
   - **Просмотр** - открыть карточки для просмотра/печати
   - **Удаление** - удалить из базы данных

### 6. Генератор контрольных работ (НОВОЕ!)

**Контрольные работы** позволяют составлять работы с задачами из **разных тем** в одном документе.

#### Создание структуры контрольной работы:

1. Откройте раздел **"Контрольные работы"** в меню
2. Нажмите **"Добавить блок задач"**
3. Для каждого блока настройте:
   - **Тема** (обязательно) - выберите тему ЕГЭ
   - **Количество задач** - сколько задач из этой темы включить
   - **Подтемы** (опционально) - уточните подтемы
   - **Сложность** (опционально) - выберите уровни: 1, 2, 3, 4 или 5
   - **Теги** (опционально) - дополнительная фильтрация
   - **Источник и год** (опционально) - например, "Решу ЕГЭ 2024"

#### Пример структуры:
```
Блок 1: Тема №5 (Показательные уравнения) - 2 задачи, сложность [1, 2]
Блок 2: Тема №14 (Стереометрия) - 3 задачи, сложность [2, 3]
Блок 3: Тема №17 (Финансовая математика) - 1 задача, сложность [3]
```

#### Генерация вариантов:

1. Настройте параметры генерации:
   - **Количество вариантов** (1-50)
   - **Режим вариантов**:
     - **Разные задачи** - каждый вариант с уникальными задачами
     - **Одинаковые, разный порядок** - задачи перемешиваются
     - **Одинаковые задачи** - все варианты идентичны
   - **Сортировка задач**: случайная, по коду, по сложности

2. Настройте формат печати:
   - **Колонки** (1-3)
   - **Размер шрифта** (10pt-20pt)
   - **Место для решения** (нет, мало, средне, много)
   - **Компактный режим** - для максимального количества задач
   - **Скрыть префиксы** - убрать "Вычислите:", "Найдите значение" и т.д.
   - **Поля для ФИО** - фамилия и имя ученика
   - **Лист с ответами** - отдельная страница с ответами

3. Нажмите **"Сформировать работу"**

#### Редактирование после генерации:

- **Drag & Drop** - перетаскивайте задачи между вариантами
- **Замена задачи** - нажмите иконку "Заменить" на задаче
- **Редактирование** - измените текст задачи прямо в интерфейсе

### 7. Редактор работ (НОВОЕ!)

Раздел **"Редактор работ"** предназначен для поддержки уже созданных работ:

1. **Список работ слева**:
   - Поиск по названию
   - Фильтры по теме и статусу (активные/архив)
   - Метрики: количество сессий, попыток и средний результат
2. **Редактирование справа**:
   - Вкладка **Состав**: управление вариантами и задачами (drag & drop, замена, удаление)
   - Вкладка **Параметры**: название, тема, лимит времени
   - Вкладка **Результаты**: выдача и итоги по ученикам
3. **Архивирование**:
   - Кнопка **Архив** переносит работу в архив (не удаляет)
4. **Сохранение**:
   - Если по работе есть попытки — доступно только **"Сохранить как новую"**
5. **Редактирование задач**:
   - Изменения применяются **глобально** ко всем работам, где используется задача

#### Сохранение и экспорт:

- **Сохранить работу** - сохранить в базу данных для повторного использования
- **Открыть сохранённую** - загрузить ранее созданную работу
- **Печать** - напечатать через браузер
- **Сохранить PDF** - экспортировать в PDF файл

#### Предпросмотр структуры:

В панели справа отображается:
- Количество блоков задач
- Количество уникальных тем
- Общее количество задач
- Временная шкала структуры (Timeline)

## 💾 Резервное копирование базы данных

> **Платформа:** macOS (LaunchAgent). Для Linux потребуется адаптация под cron/systemd.

Проект включает автоматическое ежедневное резервное копирование базы данных PocketBase.

### Что бэкапится
- `pb_data/data.db` — основная база (задачи, темы, теги, карточки)
- `pb_data/auxiliary.db` — служебная база PocketBase
- `pb_data/storage/` — файловое хранилище (если есть файлы)

### Как это работает
- Скрипт `backup.sh` использует `sqlite3 .backup` для безопасного копирования (работает даже при запущенном PocketBase)
- Бэкапы сжимаются в `.tar.gz` и сохраняются в папку `backups/`
- Автоматическая ротация: хранятся **последние 10 бэкапов**, старые удаляются
- LaunchAgent запускает бэкап **каждый день в 02:00**
- Если Mac был в режиме сна — бэкап выполнится при пробуждении
- Логи записываются в `backups/backup.log`

### Настройка автобэкапа (macOS)

```bash
# 1. LaunchAgent уже создан при установке:
# ~/Library/LaunchAgents/com.egetasks.backup.plist

# 2. Активировать (если ещё не активирован):
launchctl load ~/Library/LaunchAgents/com.egetasks.backup.plist

# 3. Деактивировать:
launchctl unload ~/Library/LaunchAgents/com.egetasks.backup.plist

# 4. Проверить статус:
launchctl list | grep egetasks
```

### Ручной запуск

```bash
./backup.sh
```

### Восстановление из бэкапа

```bash
# 1. Остановите PocketBase
# 2. Распакуйте нужный бэкап:
tar -xzf backups/backup_2026-02-01_02-00-00.tar.gz

# 3. Замените файлы БД:
cp backup_2026-02-01_02-00-00/*.db pocketbase/pb_data/

# 4. Запустите PocketBase заново
cd pocketbase && ./pocketbase serve
```

### Изменение параметров

В файле `backup.sh`:
- `MAX_BACKUPS=10` — количество хранимых бэкапов
- Время запуска — в файле `~/Library/LaunchAgents/com.egetasks.backup.plist` (поле `Hour`/`Minute`)

## 🐛 Частые проблемы и решения

### Проблема: Ошибка CORS при работе с API

**Решение:**
1. Убедитесь, что PocketBase запущен
2. Проверьте URL в файле `.env`
3. Включите анонимный доступ в настройках коллекций

### Проблема: Формулы LaTeX не отображаются

**Решение:**
1. Проверьте импорт CSS: `import 'katex/dist/katex.min.css'`
2. Убедитесь, что формулы заключены в `$...$` для inline или `$$...$$` для блочных

### Проблема: Карточки некорректно печатаются

**Решение:**
1. Используйте Chrome или Firefox для печати
2. Установите масштаб 100%
3. Выберите формат бумаги A4
4. Отключите колонтитулы в настройках печати

### Проблема: Дубликаты задач при загрузке

**Решение:**
Парсер автоматически проверяет дубликаты по полю `statement_md`. Если задача уже существует - она будет пропущена.
