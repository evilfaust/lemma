# Lemma — Платформа для учителя математики

## Описание проекта

**Lemma** — полнофункциональная веб-платформа для учителей математики: управление базой задач, генерация вариантов контрольных и самостоятельных работ, онлайн-тестирование учеников, аналитика результатов, модуль геометрии с GeoGebra, система ТДФ (теоремы, определения, формулы) и мотивация через достижения.

Приложение состоит из **трёх интерфейсов**:
- **Учительский** — управление задачами, генерация работ, выдача тестов, просмотр результатов
- **Ученический** — прохождение тестов, личный кабинет, прогресс, коллекция достижений
- **Лендинг** (`/landing.html`) — промо-страница с описанием возможностей платформы

## Основные возможности

### Управление задачами
- Просмотр и фильтрация задач по темам, подтемам, тегам, сложности, источникам, годам, наличию изображений
- Поддержка LaTeX (KaTeX) и Markdown (GFM) для математических формул и форматирования
- Редактирование задач в интерфейсе (условие, ответ, решение, пояснение, метаданные)
- Три способа добавления изображений: по ссылке, загрузка с компьютера, рисование в GeoGebra
- Обрезка изображений (слайдеры по 4 сторонам, визуальный предпросмотр)
- Система тегов с цветовой маркировкой
- Массовые операции: добавление/замена тегов, изменение сложности, источника, темы, подтем, удаление
- Импорт задач из Markdown-файлов и sdamgia.ru
- Создание задач через UI
- Уникальные коды задач (формат: `14-001`, `M17-003`)

### Модуль геометрии
- Отдельный раздел задач геометрии с фильтрами по теме/подтеме/сложности
- Два режима списка: `Таблица` и `Карточки` (переключатель в интерфейсе)
- Быстрый просмотр задачи в модальном окне с редактированием макета (`Дано` и `Чертёж`)
- Редактор GeoGebra: сохранение состояния `.ggb` + экспорт PNG-чертежа с кропом
- PNG-чертеж хранится в PocketBase storage как file field (`geogebra_image_base64`)
- Просмотр листа A5 (6 задач): ручная компоновка, drag&drop смены порядка задач, сквозная нумерация

### Модуль ТДФ (Теоремы, Определения, Формулы)
Отдельный инструмент для работы с теоретическим материалом геометрии в форме структурированных конспектов и опросников.

- **Наборы ТДФ** — организация по классам (8, 9, 10, 11 и т.д.), CRUD через TDFManager
- **Редактор конспекта** (TDFEditor) — список пунктов с drag&drop, секции-заголовки для группировки
- **Типы пунктов** (7): Теорема, Определение, Формула, Аксиома, Свойство, Признак, Следствие — с цветовой маркировкой
- **Редактор пункта** (TDFItemModal) — название, вопрос (MD+LaTeX), формулировка (MD+LaTeX), краткая запись, чертёж (PNG или GeoGebra)
- **Варианты опросников** (TDFVariantBuilder) — выбор пунктов из набора, нумерация вариантов
- **Печать** (TDFPrintView) — A4 landscape **или portrait** (переключатель), два режима:
  - `etalon` — полный конспект со всеми формулировками (5 колонок: №+тип, Тема/Формулировка, Вопрос, Чертёж, Краткая запись)
  - `blank` — бланк ученика с пустыми линиями для ответов
  - Книжный режим: компактный шрифт 9pt, умещается больше строк на странице
- Экспорт в PDF через Puppeteer-сервис

### Генератор карточек
- Случайная выборка задач по критериям
- Три формата печати: A6 (4 на лист), A5 (2 на лист), A4 (1 на лист)
- Автоматическая генерация листа с ответами
- Сохранение карточек в БД
- Настраиваемые размеры шрифта: 10–21px

### Генератор «Устный счёт»
- Распределение задач по тегам или сложности с точным контролем количества
- Компактный режим для максимального количества задач на листе
- Grid layout: 1–3 варианта на одной странице A4
- Drag & Drop: перетаскивание задач внутри и между вариантами
- Фильтр префиксов: скрытие типовых фраз («Вычислите:», «Найдите значение выражения» и т.д.)
- Горизонтальная раскладка задач
- Автоматическая прогрессия сложности
- PDF экспорт (Puppeteer + html2pdf.js fallback)
- Размеры шрифта: 10pt, 12pt, 14pt, 16pt, 20pt

### Генератор контрольных работ
- Конструктор структуры с блоками задач из разных тем
- Независимые фильтры для каждого блока (тема, подтемы, сложность, теги, источник, год)
- Три режима генерации: разные задачи / одинаковые с перемешиванием / одинаковые
- Прогрессия сложности (опционально)
- Drag & Drop задач между вариантами
- Перемещение задач из одного варианта в другой
- Замена и редактирование задач прямо в варианте
- Сохранение/загрузка работ из БД
- Экспорт в PDF
- Настройки: колонки (1–3), шрифт (10–20pt), компактный режим, место для решения, поля для ФИО, лист ответов

### Генератор вариантов ЕГЭ (EgeVariantGenerator)
Генератор полных вариантов ЕГЭ базового уровня (21 задание) в официальном стиле КИМ.

- 21 слот по темам ЕГЭ — каждый можно настроить: подтема, сложность, теги
- Возможность «закрепить» конкретную задачу в любой слот
- **Два режима печати**: обычный вариант или официальный стиль КИМ
- **КИМ-формат**: A5 portrait страницы, обложка с инструкцией, нумерация в квадратах, поле «Ответ:», футер «© Лемма»
- **Двухфазный рендер** для точной пагинации: DOM-измерение высот задач → жадное размещение по A5-страницам
- Для A4-буклета: «2 страницы на листе» в диалоге печати браузера
- Колонка «Успеваемость» в слот-таблице: средний процент верных ответов по задачам темы
- PDF-экспорт через Puppeteer-сервис

### QR-листы (QRWorksheetGenerator)
Уникальный формат работы: ученик решает задачи, находит числа-ответы в числовой таблице и закрашивает соответствующие клетки — из них складывается QR-код, который ведёт на следующее задание или ресурс.

- Выбор 3–6 задач из базы, числовые ответы образуют «правильные» клетки QR-сетки
- Ввод URL для QR-кода вручную или выбор из «Моих работ» (автоматически подставляется ссылка на сессию)
- Сокращение длинных URL через **clck.ru** (кнопка «Сократить», без прокси)
- Редактирование задачи прямо в листе (карандаш → `TaskEditModal`) без перехода в другой раздел
- Сохранение/загрузка листов в базе данных (`qr_worksheets`)
- Печать A4: страница ученика (условия + числовая сетка) + страница-ключ учителя (ответы + QR-код + подсвеченная сетка)
- Картинки к задачам: обтекание текстом (`float: right`)
- Режим **двух колонок** для листов с большими изображениями

### Пиксель-арты (PixelArtWorksheet)
Нестандартный формат работы: ученик решает задачи и закрашивает клетки сетки по числовым ответам — из них складывается пиксельный рисунок.

- Загрузка любого изображения → автоматическая бинаризация в пиксельную матрицу
- Настройка размера сетки (столбцы × строки) с сохранением пропорций
- Порог бинаризации (0–255) для тонкой настройки контраста
- Выбор задач из базы, числовые ответы образуют «правильные» клетки рисунка
- **Страница ученика**: сетка с числами + условия задач (опционально в 2 колонки)
- **Страница-ключ учителя**: подсвеченная сетка + правильные ответы (опционально)
- Режим печати на **половине листа** (2 рисунка на A4)
- Сохранение/загрузка в базе данных (`pixel_art_worksheets`), кнопка «Обновить» при повторном сохранении

### Марафон устной защиты задач (MarathonGenerator)
Интерактивный формат урока: каждый ученик получает карточку с задачей и устно объясняет решение учителю.

- Подбор задач из каталога, расстановка по сложности
- **Карточки учеников** — 6 карточек на листе A4 (сетка 2×3), цветная шапка по сложности, условие с KaTeX, логотип Lemma
- **Лист учителя** — компактная таблица: «Условие и ответ» (1/3 листа) + «Решение / Подсказка» (2/3 листа); задачи не разрываются при переносе страниц
- **Бумажный рейтинговый бланк** — горизонтальная таблица (A4 landscape): ученики × задачи, в каждой ячейке кружки для попыток и поле для очков; столбец «Итого»
- **Цифровой трекер** — интерактивная таблица в браузере: кнопки ✓/✗ для каждой ячейки, автосохранение прогресса в БД, сортировка по баллам, бейдж 🥇 для лидера

**Система очков:**

| Результат | Очки |
|-----------|------|
| Решено с первой попытки | **+3** |
| Решено со второй попытки | **+2** |
| Решено с третьей попытки | **+1** |
| Три неудачные попытки | **0** |

Сохранение/загрузка марафонов в базе данных (`marathons`). До 30 учеников и неограниченное количество задач.

### Единичная окружность (UnitCircleGenerator)
Генератор печатных листов по теме «Единичная окружность» для уроков тригонометрии.

- **Прямая задача** — точки отмечены на окружности, ученик подписывает соответствующие углы
- **Обратная задача** — дан список углов, ученик самостоятельно отмечает точки на пустой окружности
- **Смешанный режим** — чередование прямой и обратной задачи в одном варианте
- 16 стандартных позиций (кратные π/6, π/4, π/3, π/2)
- Смещение ±k×2π (k до ±4): углы вида 13π/6, −11π/6, 25π/6 и т.д.
- Все дроби отображаются через KaTeX — числитель над знаменателем
- Настраиваемые подписи: ничего / только оси (0, π/2, π, 3π/2) / все 16 позиций
- Опциональные засечки на 16 стандартных позициях и градусные подписи осей
- Форматы: **2 кружка на A4** (один под другим) или **4 кружка на A4** (сетка 2×2)
- **Страница-ключ учителя** с ответами (опционально)
- До 30 вариантов; сохранение/загрузка в базе данных (`unit_circle_worksheets`)

### Редактор работ (WorkEditor)
- Список сохранённых работ с поиском и фильтрами (тема, статус: активные/архив)
- Метрики: количество сессий, попыток, средний результат
- Редактирование вариантов: добавление/удаление/перетаскивание задач
- Замена и редактирование отдельных задач
- Перемещение задач между вариантами
- Управление параметрами работы (название, тема, лимит времени)
- Архивирование работ (без удаления)
- Панель выдачи: создание сессии, QR-код, ссылка для учеников
- Просмотр результатов учеников с возможностью выдачи достижений
- Защита: при наличии попыток — только «Сохранить как новую»

### Система тестирования учеников
- **Главная страница** (`/student/`) — личный кабинет: поле ввода кода сессии, кнопки «Войти» / «Зарегистрироваться», при входе — «Мой прогресс» / «Мои достижения»
- Учитель создаёт сессию из работы и получает ссылку `/student/{sessionId}`
- Ученик открывает ссылку, регистрируется или входит (логин/пароль)
- Ученик получает вариант (round-robin распределение)
- Ответы автосохраняются в localStorage
- Автоматическая проверка ответов:
  - Числовое сравнение с epsilon (1e-6)
  - Поддержка дробей (`1/3`), LaTeX-дробей (`\frac{1}{3}`)
  - Альтернативные ответы через `|`
- Результат показывается сразу после отправки
- Повторная попытка — только через учителя
- **Светлая и тёмная тема**: кнопка 🌙/☀️ в верхней панели, тема сохраняется в localStorage (по умолчанию — светлая)
- **Шрифт Inter**: единый стиль с лендингом

### Система достижений
- 72 предустановленных достижения с иконками
- **Случайные бейджи** (один за попытку):
  - Legendary (90%+ правильных)
  - Rare (70–89%)
  - Common (40–69%)
- **Условные достижения** (автоматическая разблокировка):
  - По результату: 100%, 90%+
  - По скорости: за 5 и за 3 минуты
  - По количеству: 1-я, 3-я, 10-я, 50-я и 100-я попытка
  - Специальные: ночная сова (после 22:00), ранняя пташка (до 08:00), воин выходных (сб/вс)
- Галерея достижений для ученика
- Учитель может выдать достижения вручную
- Включение/выключение достижений на уровне сессии
- Polling каждые 10 секунд для обновления выданных учителем наград

### Авторизация учеников
- Регистрация: имя + логин + пароль
- Авторизация через PocketBase Auth
- Привязка гостевых попыток к аккаунту при авторизации
- Панель прогресса: учитель видит всех зарегистрированных учеников, их попытки, средний балл, класс
- Детали ученика: динамика результатов и блок «Проблемные задачи» (компактный список, просмотр условия/изображения, блок сворачивается под кат)

### Статистика и аналитика
- Дашборд: распределение задач по темам, подтемам, тегам, сложности, источникам
- Облако тегов с переходом к задачам
- Поиск «белых пятен» в базе
- Кнопка перехода в «Каталог» прямо из раздела «Аналитика»
- Каталог справочников (CRUD, обнаружение и объединение дублей)

### Модуль теории
- Просмотр теоретических статей по категориям
- Редактирование статей (Monaco Editor) с поддержкой Markdown + LaTeX
- Связь теория → задачи по тегам
- Категории с CRUD
- Конструктор конспектов для печати с выбором статей и категорий

### PDF экспорт
- **Основной метод**: Puppeteer (headless Chrome) — микросервис на порту 3001
- **Запасной метод**: html2pdf.js (автоматическое переключение)
- Высокое качество рендеринга KaTeX, векторный текст
- 2–3x меньший размер файлов по сравнению с html2pdf.js

### Импорт задач из sdamgia.ru
- Парсинг задач с сайта sdamgia.ru через PDF-сервис
- Автоматическая конвертация HTML-таблиц в Markdown
- Очистка LaTeX-формул из русскоязычных alt-текстов изображений

## Структура базы данных

### Основные коллекции

#### topics — Темы ЕГЭ
```javascript
{
  id, title, section: "Алгебра" | "Геометрия",
  ege_number: number,  // Номер задания ЕГЭ (1-27)
  order, slug
}
```

#### subtopics — Подтемы
```javascript
{
  id, name, topic: relation → topics, order
}
```

#### tags — Теги
```javascript
{
  id, title, color: string  // hex
}
```

#### tasks — Задачи
```javascript
{
  id, code: string,          // "14-001", "M17-003"
  topic: relation → topics,
  subtopic: relation[] → subtopics,  // множественная
  tags: relation → tags,
  difficulty: number,        // 1 (базовый), 2 (средний), 3 (продвинутый)
  statement_md: text,        // Условие (Markdown + LaTeX)
  answer: string,
  solution_md: text,         // Решение
  explanation_md: text,      // Пояснение
  source: string, year: number,
  image: file, image_url: string,
  has_image: boolean
}
```

### Коллекции геометрии (актуально для `task-ege.oipav.ru`)

Подробная и проверенная схема production: **`SCHEMA_GEOMETRY.md`**.

#### geometry_topics — Темы геометрии
```javascript
{
  id, title, order
}
```

#### geometry_subtopics — Подтемы геометрии
```javascript
{
  id, title, topic: relation -> geometry_topics, order
}
```

#### geometry_tasks — Геометрические задачи
```javascript
{
  id, code, title,
  topic: relation -> geometry_topics,
  subtopic: relation -> geometry_subtopics,
  difficulty: number(1-3),
  task_type: "ready" | "build" | "mixed",   // optional
  statement_md, answer, solution_md,
  geogebra_base64: text,                     // .ggb состояние для повторного редактирования
  geogebra_appname: "geometry"|"graphing"|"classic"|"3d",
  geogebra_image_base64: file,               // PNG-чертеж (хранится в PocketBase storage)
  drawing_view: "image" | "geogebra",
  preview_layout: json,                      // например: { print: { image, text }, student?: { ... } }
  hints: json,                               // обычно [] если подсказки не заполнялись
  source, year, created, updated
}
```

PNG URL:
`{PB_URL}/api/files/geometry_tasks/{id}/{geogebra_image_base64}`

#### tdf_sets — Наборы ТДФ
```javascript
{
  id, title, description, class_number: number, created, updated
}
```

#### tdf_items — Пункты ТДФ
```javascript
{
  id, tdf_set: relation → tdf_sets,
  order: number, is_section_header: bool, section_title: text,
  type: "theorem"|"definition"|"formula"|"axiom"|"property"|"criterion"|"corollary",
  name: text, question_md: text, formulation_md: text, short_notation_md: text,
  drawing_image: file, geogebra_base64: text, geogebra_appname: text,
  created, updated
}
```

#### tdf_variants — Варианты опросников
```javascript
{
  id, tdf_set: relation → tdf_sets, number: number, title: text,
  item_ids: json,  // [id1, id2, ...]
  created, updated
}
```

#### qr_worksheets — QR-листы
```javascript
{
  id, title,
  qr_url: text,                  // URL для QR-кода
  tasks: relation[] → tasks,
  custom_answers: json,          // {taskId: string} — переопределённые ответы
  grid: json,                    // Cell[][] — сетка с числами
  qr_size: number                // сторона QR-матрицы (21-40)
}
```

#### pixel_art_worksheets — Пиксель-арты
```javascript
{
  id, title,
  tasks: relation[] → tasks,
  custom_answers: json,          // {taskId: string}
  grid: json,                    // Cell[][] — сетка с числами-ответами
  matrix: json,                  // boolean[][] — булевая матрица пикселей
  grid_cols: number, grid_rows: number,
  threshold: number,             // порог бинаризации (0-255)
  two_sheets: bool,              // режим полулиста
  show_teacher_key: bool,        // показывать страницу-ключ
  two_columns: bool              // задачи в две колонки
}
```

#### marathons — Марафоны
```javascript
{
  id, title,
  class_number: number,
  tasks: relation[] → tasks,
  task_order: json,           // [id, ...] — порядок задач по сложности
  students: json,             // [{name: string}] — список участников
  tracking_data: json         // {"Иванов": {"0": {attempts, solved, failed}, ...}}
}
```

#### cards — Сохранённые карточки
```javascript
{
  id, title, tasks: relation[] → tasks,
  show_answers, show_solutions, format: "А6"|"А5"|"А4",
  layout: "one-column"|"two-column", note
}
```

#### works — Контрольные/самостоятельные работы
```javascript
{
  id, title, class: number,
  topic: relation → topics,
  time_limit: number  // минуты
}
```

#### variants — Варианты работ
```javascript
{
  id, work: relation → works,
  number: number, tasks: relation[] → tasks,
  order: json  // [{taskId, position}]
}
```

### Коллекции теории

#### theory_categories
```javascript
{ id, title, description, order, color }
```

#### theory_articles
```javascript
{
  id, title, category: relation → theory_categories,
  content_md: text,  // до 100000 символов
  order, summary, tags: json, theme_settings: json
}
```

### Коллекции тестирования

#### students — Учеников (Auth-коллекция)
```javascript
{
  id, username, password (hashed),
  name: string,           // Имя (обязательное)
  student_class: string   // Класс (опционально, до 20 символов)
}
```

#### work_sessions — Сессии тестирования
```javascript
{
  id, work: relation → works,
  is_open: boolean,
  host_override: string,
  achievements_enabled: boolean,
  student_title: string  // Заголовок для стартового экрана
}
```

#### attempts — Попытки учеников
```javascript
{
  id, session: relation → work_sessions,
  student: relation → students,  // опционально
  student_name, device_id,
  variant: relation → variants,
  status: "started"|"submitted"|"corrected",
  score: number, total: number,
  achievement: relation → achievements,         // случайный бейдж
  unlocked_achievements: relation[] → achievements, // условные
  duration_seconds: number,
  submitted_at, corrected_at
}
```

#### attempt_answers — Ответы по задачам
```javascript
{
  id, attempt: relation → attempts,
  task: relation → tasks,
  answer_raw, answer_normalized: number,
  is_correct: boolean
}
```

#### achievements — Достижения
```javascript
{
  id, code: string,      // уникальный, a-z0-9_
  title, description, icon: string,
  type: "random"|"condition",
  rarity: "common"|"rare"|"legendary",
  condition_type: "score"|"speed"|"count"|"special",
  condition_value: json, order
}
```

## Установка и запуск

### Требования
- Node.js >= 18.0.0
- Python 3.8+ (для парсеров)
- npm

### Быстрый старт

```bash
# 1. Установить зависимости
npm run install:all

# 2. Создать .env файл
echo "VITE_PB_URL=https://task-ege.oipav.ru" > ege-tasks/.env

# 3. Запустить все сервисы
./start.sh
```

Или через npm:
```bash
npm run dev          # Frontend (VPS backend)
npm run migrate:task-images # Перенос внешних image_url в files tasks.image (требует PB_SUPERUSER_EMAIL/PASSWORD)
```

| Сервис | URL (Dev) | URL (Production) | Описание |

## Резервное копирование

### Создание бэкапа
```bash
./backup.sh
```

Бэкап использует `sqlite3 .backup` (безопасно при работающем PocketBase), сжимается в `.tar.gz`, хранит 10 последних копий.

### Восстановление
```bash
./restore.sh
```

Безопасная процедура с двойным подтверждением и аварийным бэкапом перед восстановлением.

### Автоматическое резервное копирование (VPS)

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| CORS ошибки | Проверить, что VPS PocketBase доступен и URL в `.env` верный |
| LaTeX не рендерится | Проверить `import 'katex/dist/katex.min.css'`; формулы в `$...$` или `$$...$$` |
| PDF сервис недоступен | Проверить доступность `https://task-ege.oipav.ru/pdf/health` |
| Карточки некорректно печатаются | Chrome/Firefox, масштаб 100%, A4, без колонтитулов |
| Дубликаты при импорте | Парсер проверяет по `statement_md`; дубли пропускаются |
| Ученик не видит тест | Проверить, что сессия открыта (`is_open: true`) |
