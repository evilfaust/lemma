# Подробное описание расширения функционала: система прохождения тестов

## Цель
Добавить в приложение полноценный режим прохождения контрольных работ учениками с мобильных телефонов. Учитель формирует и сохраняет работу, затем выдаёт ссылку/QR-код. Ученики открывают страницу, вводят ФИО, проходят тест, отправляют ответы и получают результат. После первичной проверки у ученика есть одна возможность исправить ошибки. Учитель видит все попытки, результаты, варианты и ошибки. Приём ответов можно закрыть.

## Ключевые ограничения и договорённости
- Выдача возможна только для **сохранённых** работ.
- Авторизации нет, всё работает локально в одной Wi‑Fi сети.
- На входе ученик вводит только **Фамилию Имя** (одна строка).
- Варианты распределяются **случайно**, без повторов, пока не закончатся.
- При обновлении страницы ученик получает **тот же вариант**, если ещё не отправлял.
- Кнопка «другой вариант» у учителя создаёт **новую попытку**, старую сохраняем.
- Ограничений по времени нет, но учитель может **закрыть приём**.
- Ответы — числа (целые/десятичные/дроби). Допуск сравнения: `1e-6`.
- Экспорт результатов пока не нужен.

---

## Общий пользовательский сценарий

### Учитель
1. Создаёт контрольную работу в генераторе и **сохраняет** её.
2. В блоке «Выдача» включает приём, указывает host/IP и получает ссылку/QR.
3. Раздаёт QR/ссылку ученикам.
4. После завершения видит таблицу результатов, варианты, ошибки, даты.
5. При необходимости выдаёт ученику «другой вариант».
6. Может закрыть приём — ученикам показывается сообщение о закрытии.

### Ученик
1. Сканирует QR или открывает ссылку.
2. Вводит ФИО и начинает прохождение.
3. Отвечает на задачи (можно пропускать). Отправляет.
4. Видит результат «X из Y» и список **ошибочных** задач.
5. Имеет **одну** попытку исправить ошибки.
6. После исправления видит финальный результат.

## UI/UX и страницы

### 1. Блок «Выдача» в генераторе работ
**Назначение:** сделать сохранённую работу доступной ученикам.

**Элементы управления:**
- Переключатель: «Приём ответов открыт/закрыт».
- Поле `Host` (ручной ввод IP/домена).
- Кнопки:
  - `Показать QR`
  - `Скопировать ссылку`
  - `Скачать QR`

**Ссылка:**
`http://{host_override}/student/{sessionId}`

---

### 2. Страница входа ученика
**Маршрут:** `/student/:sessionId`

**Сценарии:**
- Если `is_open = false` → показать **грозное сообщение** о закрытии приёма.
- Если `is_open = true` → форма ввода ФИО и кнопка «Начать».

---

### 3. Страница прохождения теста (мобильная)
- Компактный и удобный мобильный интерфейс.
- Список задач по порядку варианта.
- Поле ввода для каждой задачи.
- Ответы не обязательны.

Кнопка: `Отправить`.

---

### 4. Результат и исправление ошибок
После отправки:
- Показать результат «X из Y».
- Показать **только ошибочные** задачи с условиями.
- Разрешить **одну** правку неверных ответов.
- После повторной отправки — финальный результат и блокировка дальнейших изменений.

## Логика распределения вариантов

### Правила
- Варианты выбираются **случайно**.
- Пока есть невыданные варианты — не повторять.
- Если вариантов меньше, чем учеников — разрешить повторения.
- Если ученик не отправил ответы, при повторном входе получает **тот же вариант**.
- Кнопка «другой вариант» создаёт **новую попытку** и назначает новый вариант.

---

## Проверка ответов

### Допустимые форматы
- `1,5`
- `1.5`
- `3/2`
- `-2`

### Нормализация
- Запятая → точка.
- Если дробь `a/b` → вычислить как `a/b`.
- Иначе `parseFloat`.

### Сравнение
- Нормализуем и ответ ученика, и эталон.
- `Math.abs(student - correct) <= 1e-6`.

Пустой ответ = неверно.

## Данные, которые нужно хранить

### Выдача работы
- Какая работа выдана
- Открыт/закрыт приём
- Хост (IP/домен)

### Попытка ученика
- ФИО
- Назначенный вариант
- Статус (начал / отправил / исправил)
- Баллы, всего задач
- Даты начала и отправок
- Было ли исправление

### Ответы
- Сырой ввод
- Нормализованное значение
- Верно/неверно

## Коллекции (PocketBase)

### `work_sessions`
- `work` (relation → `works`)
- `is_open` (bool)
- `host_override` (text)
- `created`, `updated`

### `attempts`
- `session` (relation → `work_sessions`)
- `student_name` (text)
- `variant` (relation → `variants`)
- `status` (select: `started`, `submitted`, `corrected`)
- `score` (number)
- `total` (number)
- `correction_used` (bool)
- `created`, `submitted_at`, `corrected_at`

### `attempt_answers`
- `attempt` (relation → `attempts`)
- `task` (relation → `tasks`)
- `answer_raw` (text)
- `answer_normalized` (number)
- `is_correct` (bool)
- `checked_at` (date)
