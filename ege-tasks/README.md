# 🚀 Полная инструкция по настройке проекта

## 1. Создание проекта

```bash
# Создаем проект с Vite
npm create vite@latest ege-tasks -- --template react

# Переходим в папку
cd ege-tasks

# Устанавливаем зависимости
npm install antd pocketbase katex react-katex @ant-design/icons

# Устанавливаем dev зависимости
npm install -D @vitejs/plugin-react
```

## 2. Структура проекта

Создайте следующую структуру:

```
ege-tasks/
├── src/
│   ├── components/
│   │   ├── TaskList.jsx
│   │   ├── TaskFilters.jsx
│   │   ├── TaskCard.jsx
│   │   ├── MathRenderer.jsx        # Рендерер LaTeX формул
│   │   ├── PrintableWorksheet.jsx
│   │   └── WorksheetGenerator.jsx
│   ├── services/
│   │   └── pocketbase.js
│   ├── App.jsx
│   ├── App.css
│   └── main.jsx
├── .env
├── package.json
└── vite.config.js
```

## 3. Создайте файлы

Скопируйте все файлы из артефактов в соответствующие папки:

1. **src/services/pocketbase.js** - сервис для работы с API
2. **src/App.jsx** - главный компонент
3. **src/App.css** - стили
4. **src/main.jsx** - точка входа
5. **src/components/TaskList.jsx** - список задач
6. **src/components/TaskFilters.jsx** - фильтры
7. **src/components/TaskCard.jsx** - карточка задачи
8. **src/components/MathRenderer.jsx** - рендерер LaTeX формул
9. **src/components/WorksheetGenerator.jsx** - генератор
10. **src/components/PrintableWorksheet.jsx** - печатная форма
11. **vite.config.js** - конфигурация Vite
12. **package.json** - зависимости

## 4. Настройка окружения

Создайте файл `.env` в корне проекта:

```env
VITE_PB_URL=https://task-ege.oipav.ru
```

## 5. Настройка PocketBase (ВАЖНО!)

### Включить анонимный доступ:

1. Откройте админ-панель PocketBase: `https://task-ege.oipav.ru/_/`
2. Перейдите в **Settings** → **API Rules**
3. Для коллекций `tasks`, `topics`, `subtopics`, `tags`, `cards`:
   - **List/Search**: включите анонимный доступ
   - **View**: включите анонимный доступ
   - **Create**: для `cards`, `topics`, `subtopics`, `tags` включите анонимный доступ
   - **Update**: для `cards`, `topics`, `subtopics`, `tags` включите анонимный доступ
   - **Delete**: для `cards`, `topics`, `subtopics`, `tags` включите анонимный доступ

### Или через настройки коллекций:

1. Откройте каждую коллекцию
2. Вкладка **API Rules**
3. Для **List/Search**, **View**, **Create**, **Update**, **Delete** (для cards/topics/subtopics/tags) установите правило:
   ```
   @request.auth.id = "" || @request.auth.id != ""
   ```
   (это разрешит доступ всем, включая анонимных пользователей)

## 6. Запуск

### React приложение:
```bash
cd ege-tasks
npm run dev
```

Приложение откроется на `http://localhost:5173` и будет использовать PocketBase на VPS из `VITE_PB_URL`.

## 🖨️ Печать карточек

### Формат карточек:
- **4 карточки на 1 лист A4** (формат 2x2)
- **2 вертикальные карточки на 1 лист A4** (формат 1x2)
- **3 вертикальные карточки на 1 лист A4** (формат 1x3)
- **10 заданий на карточке** с полями для ответов
- Код карточки в левом верхнем углу (например: 14-001)
- Поле для ФИО ученика

### Рекомендации по количеству:
- **10 заданий** = 1 карточка (¼ листа)
- **20 заданий** = 2 карточки (½ листа)
- **40 заданий** = 4 карточки (1 лист A4) ✨ оптимально
- **80 заданий** = 8 карточек (2 листа A4)

### Процесс печати:
1. Сгенерируйте карточки с нужными настройками
2. Нажмите кнопку "Печать"
3. В диалоге печати выберите:
   - Формат: A4
   - Ориентация: Книжная (Portrait)
   - Поля: Нет полей (или минимальные)
   - Масштаб: 100%
4. Распечатайте и разрежьте лист по количеству карточек (2, 3 или 4)

# 🏷️ Система именования карточек

## Формат названия

Карточки автоматически получают названия в формате:

```
[Название темы или работы] [Номер]
```

Номер карточки всегда состоит из **3 цифр** с ведущими нулями: `001`, `002`, `003`, и т.д.

---

## Примеры

### Если выбрана тема
```
Настройки:
  Тема: ЕГЭ-База. №14 Вычисления
  Количество карточек: 5

Результат:
  ✓ ЕГЭ-База. №14 Вычисления 001
  ✓ ЕГЭ-База. №14 Вычисления 002
  ✓ ЕГЭ-База. №14 Вычисления 003
  ✓ ЕГЭ-База. №14 Вычисления 004
  ✓ ЕГЭ-База. №14 Вычисления 005
```

### Если тема не выбрана
```
Настройки:
  Название: Контрольная работа №1
  Тема: (не выбрана)
  Количество карточек: 3

Результат:
  ✓ Контрольная работа №1 001
  ✓ Контрольная работа №1 002
  ✓ Контрольная работа №1 003
```

### Разные темы
```
Тема 1: ЕГЭ-База. №14 Вычисления
  → ЕГЭ-База. №14 Вычисления 001
  → ЕГЭ-База. №14 Вычисления 002

Тема 2: ЕГЭ-База. №15 Текстовые задачи
  → ЕГЭ-База. №15 Текстовые задачи 001
  → ЕГЭ-База. №15 Текстовые задачи 002
```

## Преимущества

### 1. Автоматическая сортировка
Карточки автоматически сортируются по названию:
```
ЕГЭ-База. №14 Вычисления 001
ЕГЭ-База. №14 Вычисления 002
ЕГЭ-База. №14 Вычисления 003
ЕГЭ-База. №15 Текстовые задачи 001
ЕГЭ-База. №15 Текстовые задачи 002
```

### 2. Удобная навигация
Легко найти нужную карточку:
- По теме: `ЕГЭ-База. №14 Вычисления`
- По номеру: `001`, `002`, `003`

### 3. Понятная структура
Сразу видно:
- К какой теме относится карточка
- Номер карточки в серии
- Можно создавать до 999 карточек по одной теме

### 4. Отображение на карточках
Название отображается в заголовке карточки:
```
┌─────────────────────────────────┐
│ ЕГЭ-База. №14 Вычисления 001  │ ← Четкое название
│ [_______________] ФИО           │
├─────────────────────────────────┤
│ 1) задание...                   │
```

# ✨ Улучшения печати и отображения карточек

## Исправлено

### 1. ✅ Название карточки при загрузке из базы
**Проблема:** При открытии карточки "003" отображалась как "001"

**Решение:** 
- Теперь при загрузке карточки из базы используется её полное сохраненное название
- Система определяет, загружена ли карточка из базы (по наличию номера в конце названия)
- Сохраняется точное название: `ЕГЭ-База. №14 Вычисления 003`

**Результат:**
```
Открываете карточку: "ЕГЭ-База. №14 Вычисления 003"
Отображается:         "ЕГЭ-База. №14 Вычисления 003" ✓
```

### 2. ✅ Отступы для печати
**Проблема:** Карточки слишком близко к краю листа, принтер не печатает

**Решение:**
- Добавлен отступ **7 мм** от краев листа A4
- Применяется ко всем форматам карточек (А6, А5, А4)
- Отступ применяется и к страницам с ответами

**Результат:**
```
Раньше:
┌─────────────────┐
│[карточка]       │ ← слишком близко к краю
└─────────────────┘

Теперь:
┌─────────────────┐
│   [карточка]    │ ← отступ 7мм со всех сторон
└─────────────────┘
```

### 3. ✅ Стиль названия карточки
**Проблема:** Название карточки было крупным и жирным

**Решение:**
- Уменьшен размер шрифта: 16px → **12px**
- Убран жирный шрифт: bold → **normal**
- Уменьшена толщина рамки: 2px → **1px**

**До:**
```
┌────────────────────────────┐
│ ЕГЭ-База. №14 Вычисления 001 │ ← крупно и жирно
```

**После:**
```
┌────────────────────────────┐
│ ЕГЭ-База. №14 Вычисления 001 │ ← компактно и аккуратно
```

### 4. ✅ Поле для ФИО
**Проблема:** Тонкая рамка поля для фамилии

**Решение:**
- Увеличена толщина рамки: 1px → **2px**
- Поле стало более заметным и удобным для заполнения

**До:**
```
┌───────────────┬─────────────┐
│ Карточка 001  │ [_________] │ ← тонкая рамка
```

**После:**
```
┌───────────────┬─────────────┐
│ Карточка 001  │ [█████████] │ ← жирная рамка
```

### 5. ✅ Номера заданий
**Проблема:** Номера заданий были жирными

**Решение:**
- Убран жирный шрифт: bold → **normal**
- Номера выглядят более естественно

**До:**
```
**1)** задание
**2)** задание
```

**После:**
```
1) задание
2) задание
```
